## 2. 사용자 스토리

### 2.1 페이지 관리 (Page Management)

#### US-001: 페이지 생성
**Actor**: 관리자 (Admin)
**As a** 관리자
**I want to** 새로운 페이지를 생성하고 URL을 설정할 수 있도록 하기
**So that** 사이트 구조를 유연하게 확장하고 각 페이지마다 고유한 URL을 가질 수 있습니다

**인수 조건**:
- [x] "페이지 생성" 버튼을 클릭할 수 있음
- [x] 페이지 제목 입력 가능
- [x] URL 슬러그 자동 생성 및 수동 편집 가능 (예: `/about`, `/products`)
- [x] 페이지 생성 후 컨테이너 편집 화면으로 이동
- [x] 생성된 페이지는 기본값으로 미발행 상태 (is_published = false)

**UI 요구사항**:
- 모달 또는 전용 폼 페이지
- 페이지 제목, 슬러그 입력 필드
- "생성" 버튼

**API 요구사항**:
```
POST /api/pages
{
  "title": "소개",
  "slug": "about"
}
```

### 2.5 게시판 관리 (Board Management)

#### US-010: 게시판 생성
**Actor**: 관리자 (Admin)
**As a** 관리자
**I want to** 새로운 게시판을 무한으로 생성할 수 있도록 하기
**So that** 다양한 주제별로 커뮤니티를 구성할 수 있습니다

**인수 조건**:
- [x] "게시판 생성" 버튼으로 모달 열기
- [x] 게시판 이름, 타입 (일반/갤러리) 입력
- [x] 읽기 권한 설정: admin only, user, public
- [x] 쓰기 권한 설정: admin only, user
- [x] 생성 후 게시판 목록에 즉시 반영

**권한 정의**:
- **읽기 권한**:
  - `admin_only`: 로그인한 admin만 조회 가능
  - `user`: 로그인한 모든 사용자 조회 가능
  - `public`: 비로그인 사용자도 조회 가능
- **쓰기 권한**:
  - `admin_only`: admin만 게시글 작성 가능
  - `user`: 로그인한 모든 사용자 작성 가능
