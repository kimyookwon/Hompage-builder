## 1. 화면 분류

| 분류 | 설명 | 예시 |
|------|------|------|
| **Public** | 공개 사이트 (비로그인 사용자도 접근) | 홈페이지, 게시판, 게시글 |

## 2. 화면 상세 정의

### S01. 공개 홈페이지 (프론트 페이지)

**화면 ID**: S01
**타입**: Public
**URL**: `/` (홈)
**접근**: 비로그인 사용자도 접근 가능

**목적**: 관리자가 구성한 컨테이너들을 렌더링하여 최종 웹사이트 표시

**주요 컴포넌트**:
1. **네비게이션 헤더**
   - 로고 (site_settings.logo_url)
   - 메뉴 (페이지 목록의 is_published=true인 항목)
   - 로그인/회원가입 또는 사용자 메뉴 (로그인 여부에 따라)
   - 다크모드 토글 버튼

2. **동적 섹션 렌더링**
   - page_sections를 순서대로 렌더링
   - 각 섹션의 format에 따라 다른 스타일 적용 (CSS 변수 사용)
   - 반응형 디자인 (모바일/태블릿/데스크톱)

3. **디자인 토큰 적용**
   - primary_color, secondary_color, background_color를 CSS 변수로 주입
   - 모든 섹션이 해당 변수 참조

4. **푸터**
   - 저작권 정보
   - 링크 (개인정보보호, 이용약관 등) - Phase 2

**데이터 요구**:
- pages (slug로 조회)
- page_sections (page_id로 모든 섹션 조회)
- site_settings (로고, 컬러, GTM 코드)
- users (현재 로그인 사용자 정보)

**관련 API**:
- `GET /public/pages/{slug}` (페이지 + 섹션 + 설정)
- `GET /auth/me` (현재 사용자 정보, 선택적)

**특이사항**:
- 메타 태그 (SEO) 적용 (Phase 2)
- GTM 코드 자동 삽입
- 모바일 네비게이션 (햄버거 메뉴)

### S02. 로그인 / 소셜 로그인

**화면 ID**: S02
**타입**: Auth
**URL**: `/login`
**접근**: 비로그인 사용자만 접근 가능 (이미 로그인하면 리다이렉트)

**목적**: 이메일/비밀번호 자체 로그인 및 소셜 로그인 제공

**주요 컴포넌트**:
1. **로그인 폼**
   - 이메일 입력 필드
   - 비밀번호 입력 필드 (마스킹)
   - "로그인" 버튼
   - "비밀번호 찾기" 링크 (Phase 2)

2. **소셜 로그인 버튼 (3개)**
   - Google 로그인 (파란색 버튼, Google 로고)
   - Kakao 로그인 (노란색 버튼, Kakao 로고)
   - Naver 로그인 (초록색 버튼, Naver 로고)
   - 각 버튼은 OAuth 제공자의 로그인 페이지로 리다이렉트

3. **회원가입 링크**
   - "아직 계정이 없으신가요? 회원가입"

4. **에러 메시지 표시**
   - 유효하지 않은 자격증명
   - 차단된 사용자

**데이터 요구**: 없음

**관련 API**:
- `POST /auth/login`
- `POST /auth/oauth/google/callback`
- `POST /auth/oauth/kakao/callback`
- `POST /auth/oauth/naver/callback`

**처리 흐름**:
```
1. 사용자가 "Google 로그인" 클릭
2. Google OAuth URL로 리다이렉트
3. 사용자 인증 후 authorization code 반환
4. 백엔드 callback 엔드포인트 호출
5. JWT 토큰 수신
6. localStorage에 토큰 저장
7. 관리자 대시보드 또는 공개 사이트로 리다이렉트
```

**특이사항**:
- 자동 이메일 입력 필드 포커스
- Enter 키로 로그인 가능
- 로딩 중 버튼 비활성화

### S03. 관리자 대시보드 (홈)

**화면 ID**: S03
**타입**: Admin
**URL**: `/admin` 또는 `/admin/dashboard`
**접근**: 관리자만 접근 가능 (JWT 토큰 + role='admin')

**목적**: 전체 현황 요약 및 빠른 링크 제공

**주요 컴포넌트**:
1. **상단 네비게이션**
   - 로고
   - 메뉴 (대시보드, 페이지, 컨테이너, 게시판, 회원, 설정)
   - 사용자 프로필 메뉴 (로그아웃)

2. **통계 카드 (4개 열)**
   - 총 방문자 수 (Phase 2)
   - 총 게시글 수
   - 총 회원 수
   - 차단된 회원 수

3. **최근 게시글 목록 (테이블)**
   - 게시판명, 제목, 작성자, 작성일, 댓글 수
   - 페이지네이션 없음 (최신 10개만)

4. **빠른 링크 (버튼)**
   - "새 페이지 생성"
   - "게시판 관리"
   - "회원 조회"

**데이터 요구**:
- 통계 (POST/USER/COMMENT 개수)
- 최근 게시글 10개

**관련 API**:
- `GET /admin/stats` (전체 통계) - 신규
- `GET /boards` (게시판 목록)
- `GET /users?limit=5` (최근 회원)

### S04. 페이지 관리

**화면 ID**: S04
**타입**: Admin
**URL**: `/admin/pages`
**접근**: 관리자만 접근 가능

**목적**: 페이지 목록 조회, 생성, 수정, 삭제

**주요 컴포넌트**:
1. **상단 액션 바**
   - "새 페이지 생성" 버튼 (모달 또는 폼 페이지)

2. **페이지 목록 테이블**
   - 칼럼: ID, 제목, URL 슬러그, 발행 여부 (토글 스위치), 섹션 수, 작성일, 액션
   - 정렬 기능 (제목, 작성일)
   - 검색 기능 (제목, 슬러그)
   - 페이지네이션

3. **액션 버튼 (각 행)**
   - "편집" (컨테이너 편집기로 이동)
   - "미리보기" (공개 페이지 미리보기)
   - "삭제" (확인 모달 후 삭제)

4. **발행 토글 스위치**
   - 클릭 시 즉시 is_published 변경

**데이터 요구**:
- pages (페이지네이션)

**관련 API**:
- `GET /pages?page=1&limit=20`
- `POST /pages` (생성)
- `DELETE /pages/{page_id}` (삭제)
- `PATCH /pages/{page_id}/publish` (발행 토글)

**특이사항**:
- 새 페이지 생성 시 기본 헤더 섹션 자동 생성
- 슬러그 자동 생성 (제목에서 파생) 및 수동 편집 가능
- 삭제 전 확인 모달 필수

### S05. 컨테이너 편집기

**화면 ID**: S05
**타입**: Admin
**URL**: `/admin/pages/{page_id}/edit`
**접근**: 관리자만 접근 가능

**목적**: 페이지의 섹션(헤더/컨테이너/배너/푸터)을 추가, 정렬, 포맷 선택, 편집

**주요 컴포넌트**:
1. **좌측 사이드바 (섹션 트리)**
   - 헤더 (고정, 1개)
   - 컨테이너 목록 (무한, 드래그 정렬)
   - 배너 (무한)
   - 푸터 (고정, 1개)
   - 각 섹션 항목: 타입 아이콘 + 이름 + 삭제 버튼

2. **메인 편집 영역**
   - 선택된 섹션의 상세 편집 폼
   - 포맷 선택 (드롭다운 또는 라디오)
   - 콘텐츠 입력 (포맷별로 다름)
   - 저장 버튼

3. **우측 미리보기 패널**
   - 현재 페이지의 실시간 미리보기
   - 반응형 뷰 토글 (모바일/태블릿/데스크톱)

4. **상단 액션 바**
   - "섹션 추가" 버튼 (모달: 타입 선택 → 포맷 선택)
   - "저장" 버튼 (전체 변경사항 저장)
   - "취소" 버튼 (변경 취소)

**데이터 요구**:
- pages (특정 페이지)
- page_sections (해당 페이지의 모든 섹션)
- site_settings (미리보기에 필요)

**관련 API**:
- `GET /pages/{page_id}`
- `GET /pages/{page_id}/sections`
- `POST /pages/{page_id}/sections`
- `DELETE /sections/{section_id}`
- `PATCH /pages/{page_id}/sections/reorder`
- `GET /site-settings` (미리보기)

**포맷별 편집 폼**:
1. **Bento Grid**
   - 제목, 설명
   - 항목 목록 (제목, 이미지, 크기)
   - 항목 추가/삭제

2. **Glassmorphism**
   - 배경 이미지 (업로드)
   - 제목, 설명
   - 카드 목록

3. **Organic Shapes**
   - 제목
   - SVG 모양 선택 (프리셋)
   - 항목 목록

4. **Text**
   - 제목, 부제목
   - 본문 (마크다운 에디터)
   - 정렬 (좌/중앙/우)

5. **Gallery**
   - 갤러리 제목
   - 이미지 목록 (업로드/URL)
   - 레이아웃 선택 (1열/2열/3열)

**특이사항**:
- 섹션 드래그 앤 드롭으로 정렬 (SortableJS 등)
- 미리보기 실시간 업데이트
- 자동 저장 (debounce, 1초)
- 실행 취소/다시 실행 (Phase 2)

### S06. 시서스 관리 (사이트 설정)

**화면 ID**: S06
**타입**: Admin
**URL**: `/admin/settings`
**접근**: 관리자만 접근 가능

**목적**: 전역 사이트 설정 (로고, 디자인 토큰, GTM, OAuth 키 등)

**주요 컴포넌트**:
1. **로고 업로드**
   - 이미지 업로드 드래그 앤 드롭 영역
   - 미리보기
   - 기존 로고 삭제 버튼
   - 파일 크기 제한: 5MB

2. **디자인 토큰 (컬러 피커)**
   - 주 컬러 (Primary) 피커
   - 보조 컬러 (Secondary) 피커
   - 배경 컬러 (Background) 피커
   - 각 피커는 HEX 입력 또는 컬러 선택 가능

3. **GTM 설정**
   - GTM Container ID 또는 전체 스크립트 입력 필드
   - 설명: "Google Tag Manager 코드를 붙여넣으세요"

4. **OAuth 설정** (Phase 2)
   - Google Client ID, Secret 입력 필드
   - Kakao API Key, Secret 입력 필드
   - Naver Client ID, Secret 입력 필드

5. **저장 버튼**
   - 모든 변경사항을 site_settings에 저장

**데이터 요구**:
- site_settings

**관련 API**:
- `GET /site-settings`
- `PATCH /site-settings`

**특이사항**:
- 변경 후 "저장" 클릭 시 즉시 공개 페이지에 반영
- 컬러 변경 시 실시간 미리보기 (하단에 샘플 표시)
- 로고 업로드 시 파일 검증 (이미지 타입, 크기)

### S07. 회원 관리

**화면 ID**: S07
**타입**: Admin
**URL**: `/admin/members`
**접근**: 관리자만 접근 가능

**목적**: 가입 회원 목록 조회, 등급 변경, 차단, 상태 관리

**주요 컴포넌트**:
1. **검색 및 필터**
   - 검색어 입력 (이메일, 이름)
   - 상태 필터 (전체, 활성, 차단)
   - 정렬 (가입일, 이름, 이메일)

2. **회원 목록 테이블**
   - 칼럼: ID, 이메일, 이름, 소셜 제공자 (뱃지), 역할, 상태, 가입일, 액션
   - 페이지네이션 (50명/페이지)

3. **소셜 제공자 뱃지**
   - Google: 파란색 뱃지 + G 로고
   - Kakao: 노란색 뱃지 + K 로고
   - Naver: 초록색 뱃지 + N 로고
   - 자체 가입: 뱃지 없음

4. **액션 버튼 (각 행)**
   - 역할 드롭다운 (admin ↔ user)
   - 상태 드롭다운 (active ↔ blocked)
   - "자세히" 버튼 (사용자 상세 정보 모달)

5. **사용자 상세 정보 모달**
   - ID, 이메일, 이름, 역할, 상태
   - 가입일, 마지막 로그인 (Phase 2)
   - 게시글 수, 댓글 수
   - 변경 내역 (로그, Phase 2)

**데이터 요구**:
- users (페이지네이션, 필터)
- posts (사용자별 게시글 수)
- comments (사용자별 댓글 수)

**관련 API**:
- `GET /users?page=1&limit=50&search=&sort=created_at`
- `PATCH /users/{user_id}/role`
- `PATCH /users/{user_id}/status`
- `GET /users/{user_id}` (상세, 신규)

**특이사항**:
- 드롭다운 변경 시 즉시 API 호출
- 변경 전 확인 모달 (역할, 상태 변경)
- 소셜 제공자는 표시만 (변경 불가)

### S08. 게시판 관리

**화면 ID**: S08
**타입**: Admin
**URL**: `/admin/boards`
**접근**: 관리자만 접근 가능

**목적**: 게시판 생성/수정/삭제, 권한 설정, 게시글 관리

**주요 컴포넌트**:
1. **상단 액션 바**
   - "새 게시판 생성" 버튼

2. **게시판 목록 테이블**
   - 칼럼: ID, 이름, 타입, 읽기 권한, 쓰기 권한, 게시글 수, 작성일, 액션
   - 검색 기능 (게시판명)
   - 정렬 (작성일, 이름)

3. **게시판 생성/수정 모달**
   - 게시판명 입력
   - 타입 선택 (일반, 갤러리)
   - 읽기 권한 선택 (admin_only, user, public)
   - 쓰기 권한 선택 (admin_only, user)
   - 저장 버튼

4. **액션 버튼 (각 행)**
   - "게시글 보기" (게시글 목록 페이지)
   - "수정" (모달에서 편집)
   - "삭제" (확인 모달 후 삭제)

5. **게시글 목록 (상세 페이지)**
   - 해당 게시판의 모든 게시글 표시
   - 제목, 작성자, 작성일, 댓글 수
   - 게시글 삭제 (관리자만)

**데이터 요구**:
- boards (페이지네이션)
- posts (게시판별 게시글 수)

**관련 API**:
- `GET /boards`
- `POST /boards` (생성)
- `PATCH /boards/{board_id}` (수정, 신규)
- `DELETE /boards/{board_id}` (삭제)
- `GET /boards/{board_id}/posts` (게시글 목록)

**특이사항**:
- 게시판 생성 후 게시글 관리 페이지로 자동 이동
- 권한 설정은 공개 페이지에 즉시 반영
- 게시판 삭제 시 모든 게시글도 함께 삭제

### S09. 공개 게시판 페이지

**화면 ID**: S09
**타입**: Public
**URL**: `/boards/{board_id}` 또는 `/boards/{board_name}`
**접근**: 게시판의 읽기 권한에 따라 결정

**목적**: 사용자가 게시글을 읽고 댓글을 작성하는 공개 페이지

**주요 컴포넌트**:
1. **게시판 헤더**
   - 게시판명
   - 게시글 수

2. **게시글 목록 (테이블 또는 카드)**
   - 제목, 작성자, 작성일, 댓글 수
   - 페이지네이션 (20개/페이지)

3. **검색 및 정렬**
   - 검색어 입력 (제목)
   - 정렬 (최신순, 인기순 - 댓글 수)

4. **게시글 작성 버튼**
   - 권한 있으면 표시 (write_permission 확인)
   - 권한 없으면 숨김 또는 비활성화

5. **게시글 상세 (클릭 시 모달 또는 페이지)**
   - 제목, 내용, 작성자, 작성일
   - 댓글 목록
   - 댓글 작성 폼 (권한 있으면)

6. **댓글 목록**
   - 댓글 내용, 작성자, 작성일
   - 자신의 댓글은 삭제 버튼

**데이터 요구**:
- boards (권한 확인)
- posts (게시판의 게시글)
- comments (게시글의 댓글)
- users (현재 로그인 사용자)

**관련 API**:
- `GET /boards/{board_id}/posts?page=1&limit=20`
- `GET /posts/{post_id}` (상세)
- `POST /boards/{board_id}/posts` (작성)
- `POST /posts/{post_id}/comments` (댓글 작성)

**권한 기반 렌더링**:
- read_permission이 "public" → 비로그인 사용자도 조회 가능
- read_permission이 "user" → 로그인 필수
- read_permission이 "admin_only" → 관리자만 조회
- write_permission이 "user" → 로그인 사용자만 작성 가능
- write_permission이 "admin_only" → 관리자만 작성 가능

**특이사항**:
- 게시글 작성 시 제목, 내용 입력
- 댓글은 계층형이 아님 (단순 선형 구조)
- 모바일에서는 테이블을 카드 레이아웃으로 변경

```
[공개 사이트]
    ↓ "로그인" 클릭
```

```
[S03: 대시보드]

    ├─ "설정" 메뉴 → [S06: 시서스 관리]
    ├─ "회원" 메뉴 → [S07: 회원 관리]
    └─ "게시판" 메뉴 → [S08: 게시판 관리]
```

### 3.3 공개 사이트 플로우

```
[S01: 홈페이지]
    ├─ 메뉴 클릭 → [S01: 다른 페이지] (동적 페이지)
    ├─ "게시판" 링크 → [S09: 공개 게시판]
    ├─ "로그인" 버튼 → [S02: 로그인]
    └─ 다크모드 토글 → [S01: 다크모드 적용]
```
